## 🚀 下一步：部署到生产环境

### 方案 A: EdgeOne Pages (前端) + Railway (后端)

#### 1. 部署后端到 Railway

```bash
# 安装 Railway CLI
npm install -g @railway/cli

# 登录
railway login

# 在后端目录部署
cd backend
railway init
railway up

# 配置环境变量（在 Railway 控制台）
# 添加所有必需的环境变量
```

**Railway 环境变量**:
- `OPENAI_API_KEY`
- `OPENAI_API_BASE`
- `OPENAI_MODEL`
- `EMBEDDING_MODEL`
- `EMBEDDING_API_BASE`
- `EMBEDDING_API_KEY`
- `MAX_HISTORY_LENGTH`

**获取后端 URL**:
```bash
railway domain
# 输出: https://three-damo-production.up.railway.app
```

#### 2. 部署前端到 EdgeOne Pages

**方法 1: 通过 Git 自动部署**

1. 登录 EdgeOne Pages 控制台
2. 点击"新建项目"
3. 选择 GitHub 仓库: `Moowningstar/tree-damo-web`
4. 配置构建设置:
   ```
   框架预设: Vite
   构建命令: bun run build
   输出目录: dist
   Node 版本: 18.x
   ```
5. 添加环境变量:
   ```
   VITE_API_BASE_URL=https://three-damo-production.up.railway.app
   ```
6. 点击"部署"

**方法 2: 手动上传**

```bash
cd frontend
bun run build
# 上传 dist/ 目录到 EdgeOne Pages
```

## 🔧 故障排查

### 问题 1: CORS 错误

**症状**: 浏览器控制台显示 CORS 错误

**解决**:
1. 确认后端 CORS 配置包含前端域名
2. 在后端添加环境变量 `FRONTEND_URL=https://your-frontend.edgeone.ai`
3. 重新部署后端

### 问题 2: 前端无法连接后端

**症状**: 网络请求失败

**解决**:
1. 检查 `VITE_API_BASE_URL` 是否正确
2. 确认后端服务正在运行
3. 测试后端 API: `curl https://your-backend.com/health`

### 问题 3: 环境变量未生效

**症状**: 使用默认值而非配置值

**解决**:
1. 确认环境变量名称正确（前端必须以 `VITE_` 开头）
2. 修改环境变量后需要重新构建
3. 检查部署平台的环境变量设置
